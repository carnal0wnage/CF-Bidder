    Options
    Back To The Top
    This page allows you the Admin of the auction website to set what functions each of your sellers will be able to perform and how they will be able to perform these actions. Below is a detailed explanation of how each of these sections work:

    SELLER FUNCTIONALITY:

    Allow sellers to email all winning bidders
    This allows the seller to e-mail all the winning bidders. This feature can be turned on or off / allowed or disallowed.

    Allow seller to email all losting bidders
    This allows the seller to e-mail all the losing bidders. This feature can be turned on or off / allowed or disallowed.

    Allow sellers to copy exisiting lots
    This feature allows the seller to copy existing lots and create new ones that are identical to the one copied. This feature can be turned on or off.

    Allow sellers to delete lots before they open
    This allows the seller to delete one of their own lots before it is set to be open. This feature can be turned on or off.

    Sellers may re-open lots that received no bids
    If a certain lot has not received a bid from it's opening time to it's closing time, turning on this feature will allow the seller to re-open that same lot.

    Sellers may re-open lots where reserve price was not met
    If a certain lot has not received a bid that meets the reserve price (the price, established by the seller before he or she has to give away the item being auctioned off.) then the seller can re-open that lot after the lot closes. This option can be turned on or off.

    SELLER & LOT APPROVAL OPTIONS:

    Allow sellers to request an e-mail with lot information after creating / editing lot:
    This function will allow the seller to request an e-mail from the system with lot information after creating or editing a lot. This feature can be turned on or off by using by either checking or unchecking the checkbox next to it.

    Require Seller to Be Approved:
    This feature lets you the Admin of the auction website decide whether any seller by default has to be approved as a seller by you or another Admin. You can turn this feature on or off.

    Send E-Mail to Admin With Seller, Lot Information
    This feature let you the Admin have the software system e-mail you the seller and lot information each time a new lot is created. This feature can be turned on or off regardless of whether each lot has to be approved or not.

    Require each lot to be approved by admin before posting:
    This feature lets the Admin decide whether each lot by default has to be approved by the Admin before it can be posted for bidding on the auction website. This feature can be turned on or off.

    Require each lot created by seller on probation be approved:
    This feature allows you to control certain sellers. This means that sellers that you have placed in a probation status, need to have all new lots that they create approved by the Admin. This feature can be turned on or off.

    E-Mail seller lot information to admin regardless of whether approval is needed:
    This means e-mail the lot information to the admin for all lots. Everytime a seller creates a lot, email the admin with the info for that lot - regardless of whether or not the lot needs to be approved.

    E-Mail seller with lot information after admin approval or rejection.
    This allows you the Admin to determine whether the seller will be e-mailed with lot information after admin approval or rejection. This feature can be turned on or off.

    Admin Name:
    This is the name of the Administrator that will be displayed as the Admin Name in the e-mails that will be sent to the seller in all of the above cases.

    Admin E-Mail Address:
    This is the e-mail address of the Administrator that will be displayed as the admin e-mail for the above e-mail communication that will be conducted with the sellers.

    SELLER HOMEPAGE:

    This section of the page allows you to define which information will be displayed on each Seller's Homepage which will define what users (bidders) are able to see.

    In order accept any of the changes that you have made on this page, you must click on the "Update Seller Options" Button.

    Back To The Top
    SELLER INSERTION FEES

    Insertion Fee:
    This is the insertion fee that will be charged to a seller each time they create a lot with an item being bid on. This insertion fee is optional, and a "0" denotes no insertion fee.

    Multiply Insertion fee by lot quantity:
    This feature allows you to multiply whatever the value of your insertion fee is by the quantity of items in a lot.

    e.g if your insertion fee is $2.00 and a seller creates a lot with a quantity of 10 items, then your insertion fee is : $20.00.

    Vary Insertion fee rather than use above amount:
    Insertion fee will vary based on the opening bid or reserve price based on a scale. If opening bid (or reserve price) is up to $X, insertion fee is $Y.

    Use Reserve price instead of opening bid:
    in the above, use the reserve price instead of opening bid as the basis of determining what the insertion fee is. so this would be a higher $X above, and thus a higher $Y since the reserve price is higher than the opening bid.

    Chart:
    This chart allows you to set varying insertion fees that are based on the amount of the opening bid.

    e.g. In the first row (1.) : If you were to enter a value of 0.50 in the "fee field" and $5.00 in the Upper range field. This would mean that if an opening bid is up to $5.00 - the insertion fee is $0.50.

    Continuing down the list, in the second row in (2.) you could enter a fee of $1 and upper range of $10. This means that if the opening bid is between $5.01 and $10 the insertion fee for that item is now $1.

    FEES FOR ADD-ON OPTIONS

    These options allow you the Admin to determine how much sellers are charges to have their lots featured and therefore become more prominent in various methods.

    Bold Lot Name:
    This feature allows you the Admin to determine how much a seller is charged to have their lot name Bolded amongst other lots in a category. If you are not offering this option fill in a "0" in the field.

    Lot Featured On Category Page:
    This feature allows you the Admin to determine how much a seller is charged to have their lot featured on a category page. If you are not offering this option fill in a "0" in the field.

    Lot featured on homepage:
    This feature allows you the Admin to determine how much a seller is charged to have their lot featured on your website auction homepage. If you are not offering this opiton fill in a "0" in the field.

    SELLER COMMISSION FEES

    This section allows you to determine what type of commission fees you will be charging sellers by default.

    % of Revenue:
    This simply allows you the Admin to determine the percentage of revenue that is made of a lot by default. The default amount is 10%

    Minimum Amount:
    This is the minimum amount of what the commission fee will be on each lot that is opened by sellers.

    Maximum Amount:
    This is the maximum amount in US dollars, of what the commission fee will not be exceeded by default on all lots opened by sellers.

    Vary commission percentage based on winning bid(s)
    Like the varying insertion fee except a percentage.

    Calculate for each bid, not total of all winning bids for lot
    For the above, are the X values the total sum of all winning bids, in which case the X will be larger; or do we calculate the proper Y based on each individual X (bid * quantity) and then add them up for all bids

    Percentages apply in step fashion
    rather than saying we get Y% of the winning bid, you get a certain percentage for each step. this is how ebay does it. for instance, you take 2% of the first $10.00 of a winning bid, 1.5% of the amount between $10.01 and $50.00, and then 1% of anything above that

    so if the winning bid were $100, you would get:
    (2% of 10) + (1.5% of 40) + (1% of 50)

    if the winning bid were $30, you would get:
    (2% of 10) + (1.5% of 20)

    CREATE LOT OPTIONS

    This section allows you the Admin to determine what type of options a seller will be able to manipulate when they are creating a lot.

    Close Basis:
    This feature allows a seller to determine the basis on which a lot will be closed on. This can be turned off or on.

    Quantity:
    This feature allows a seller to determine the quantity of units that will be going on the bidding block in a specific lot. This feature can be turned on or off.

    Opening Bid:
    This feature allows a seller to determine what the opening bid in US dollars will be on the lot that they are opening. This feature can be turned on or off.

    Sell Price:
    This feature allows a seller to determine what the sell price of an item that they are auctioning off will be. The sell price means that they can negotiate a selling to bidder for this amount. This feature can be turned on or off.

    Shipping & Handling Charge:
    This feature allows a seller to determine a shipping and handling charge that will be applied to the winning bid. This feature can be turned on or off.

    Shipping Method:
    This feature will allow a seller to determine a shipping method that they want to use to ship the item to the winning bidder. This feature can be turned on or off.

    Close Inactivity:
    This feature is part of the Close Basis. It allows the seller to determine a period of inactivity after which a lot will close, based on that period of inactivity being exceeded. This feature can be turned on or off.

    Maximum Quantity:
    Maximum quantity is the maximum number of items a bidder can win in a particular lot. say the lot has 10 items, but the seller does not want anyone to win more than 3 of them

    Reserve Price:
    This feature will allow sellers to create a reserve price. This means that a seller can reserve a price at which he or she does not have an obligation to auction of the item. This feature can be turned on or off.

    Summary:
    Summary is a shorter description field. not really used right now, but it's there if you want it

    Location:
    This feature will allow sellers to reveal the location of the item that they are putting up for bidding. This feature can be turned on or off.

    Payment Method:
    This feature will allow sellers to tell bidders what payment methods they are accepting for the lot item. This feature can be turned on or off.

    Description:
    This feature will allow sellers to create a description for the lot and the lot item. This feature can be turned on or off.

    Allow HTML tags in description:
    This feature, if turned on will allow sellers to insert HTML tags into their descriptions their lots. This is potentially a bad idea, but if you trust your users it can add to those who know how to handle HTML well to increase their potential to lure in more bidders to their lots. This feature also can be turned on or off.

    Upload Image to Server:
    If checked, this feature allows the seller to upload an image of the item they are auctioning off and display it on their lot page. This feature can be turned on or off.

    Upload thumbnail image to server:
    If turned on, this feature allows the seller to upload a thumbnail image (smaller image) to the server and display it on the lot page. This image will be clickable and will lead to the full sized image if it has been also uploaded. This feature can be turned on or off.

    Enter URL to Image:
    If turned on, this feature allows the seller to display the full sized image on his lot page by referencing it from another URL. This feature can be turned on or off.

    Enter URL to Thumbnail Image:
    If turned on, this feature allows the seller to display the thumbnail image (smaller image) of the item on the lot which links to the larger image, by referencing it from another URL. This feature can be turned on or off.

    OPEN DATE/TIME

    This section allows you the Admin to determine what type of opening date and time options sellers will have the ability to control and manipulate when they are creating a new lot.

    Choose full opening date/time:
    Choosing this will allow sellers to have full control over when they want to open their lot.

    Choose Opening Date, But at Default time:
    This option will allow sellers to have control over the opening date, but the lot will open at the default time you have specified for lots to open during the day.

    Choose Opening Time, but date is default date:
    This option will allow sellers to have control over the opening time of the lot but the lot will open on the default date as specified by the Admin for default opening dates.

    No Choice: Lot Opens Immediately (or after Admin approval)
    This option will not give a seller any choice in determining when their lot will open. The lot will open immediately or will open after the Admin has approved it if necessary.

    No Choice: Lot Opens at default date / time (or after Admin approval)
    This option will not give a seller any choice in determining when their lot will open. The lot will open at the default date and time specified by the Admin or after the Admin has approved it at the default date and time.

    Lot Opens Immediately (or after Admin approval):
    Gives seller option of "Now" in addition to above option.

    CLOSE DATE/TIME:

    This section allows you, the Admin, to determine what lot closing date and time options a seller will be able to manipulate.

    Choose Full Closing Date / Time:
    This option will give the seller full control of the closing date and time of the lot(s) that they are opening.

    Choose closing date, but at default time:
    This option will give the seller control of the closing date of a lot, but the lot will close at the default time set by the Admin in the default lot options.

    Choose Closing Time, But Date Is By Default:
    This option will give the seller control of the closing time of a lot, but the lot will close at the default date set by the Admin in the default lot options.

    No Choice. Lot closes at default date/time:
    This option will not only allow the seller to choose a number of days

    Choose Number of days (full choice):
    Keeping this option on will allow a seller to specify a number of days that the lot should be open.

    Choose number of days from a pre-defined list:
    Keeping this option on will allow a seller to select from a list the number of days that the lot should stay open.

    TEMPLATE

    A template determines how the lot is displayed. the admin can create different templates, for instance private templates, templates that display all bids, template that display just winning bids, etc. admin can decide which templates the seller may choose from

    BID INCREMENT

    This section will allow you the Admin to control how bid incriments will behave on your auction website.

    Use Default Bid Increment:
    Choosing this will have your system use the default bid incriment that you have specified in your lot options for each lot as default.

    Allow seller to choose bid increment:
    Choosing this will allow the seller to determine a bid incriment for any of his or her lots that he or she creates.

    Vary bid increment based on amount of opening bid:
    Similar to the insertion fee. basically, If the seller wants the bid increment to be higher for items that have higher opening bids, this lets you do that. If opening bid is $X, bid increment is $Y.

    Use reserve price instead of opening bid:
    This will allow sellers to choose their reserve price as an opening bid instead of an opening bid that they determine.

    Category:
    This is which category they can post items into. Useful if you don't want items in top-level categories, or want to post special items in a category.

    In order to enable any of the changes that you made on this page you must click on the "Update Seller Options" button at the bottom of the page.

    User Agreement
    Back To The Top
    This is the additional User Agreement that sellers must sign in order to be able to open lots in your auction website. This is typically a very important agreement as it may serve to protect you the website owner from liablities on what is auctioned off from your website without your direct knowledge. Below are the the fields that you can modify or add to for the Seller User Agreement:

    HTML VERSION FOR THE SITE:
    This version of the Seller User Agreement will be shown to the seller when they read the agreement. You may choose to add any HTML code here as necessary.

    TEXT VERSION FOR E-MAIL:
    This version of the Seller User Agreement will be sent via e-mail to the seller that is registering to auction off items on your website. This in effect should be the same agreement that is used on the website.

    MESSAGE WHEN A SELLER USER AGREEMENT IS ACCEPETED:
    This is the message that will appear on the webpage once the seller has agreed to abide by your Seller User Agreement.

    MESSAGE WHEN SELLER USER AGREEMENT IS REJECTED:
    This is the message that will appear on the webpage once the seller has not agreed to abide by your Sellar User Agreement.

    Create Lot Options
    Back To The Top
    This page allows you as the Admin to manage the default settings for all your lots. This means that any new lot that does not have any predetermined lot specifications created by either the Admin or the seller, will acquire these default settings:

    OPENING BID:
    This is the default opening bid for any lot any item in US dollars.

    BID INCREMENT:
    This is the default bid increment (or the amount by which each bid must be increased by) in US dollars.

    HIPPING:
    A future feature currently being developed.

    RESERVE PRICE:
    This is the default reserve price that will appear in all lots by default. This means that $5.00 USD is by default the minimum price to which a lot has to be bid on before the seller loses the right to retain the object being auctioned off.

    IMAGE:
    This is the default directory that you will store your full sized product images in for display in each lot.

    THUMBNAIL:
    This is the default directory that you will store your thumbnail sized product images in for display in each lot.

    DEFAULT LOT OPENING TIME:
    This area allows the Admin to manage the default lot opening times on all lots. The Admin can choose to make it any of the following combinations:

    a) Day of the week + Time
    b) Day of the week + Number of hours and minutes after the next hour
    c) Number of days after current date + Time
    d) Number of days after current date + number of hours and minutes after the next hour

    DEFAULT LOT CLOSING TIME:
    This area allows the Admin to manage the default lot closing times on all lots. The Admin can choose t omake it any of the following combinations:

    a) Day of the Week + Time
    b) Day of the Week + Number of hours and minutes after opening time.
    c) Number of days after opening date + Time
    d) Number of days after opening date + Number of hours and minutes after opening time

    CLOSE BASIS:
    This allows you the Admin to set a default close basis for each of your lots. This means that by default all your lots will close according to one of the following:

    a) Time
    b) Time + Inactivity (in whole minutes)

    Closing Options
    Back To The Top
    This page details some of the options in which e-mail send outs upon the closure of a lot are determined. Below is a full explanation of each section:

    AUTOMATIC E-MAILS

    This section talks about the automatic e-mails that are sent out upon closure of a lot.

    >> Automatically e-mail lot contact person to notify them that auction has closed
    (this can be turned on or off)

    >> Include list of winning bidders in e-mail
    (this includes a list of winning bidders in the automatic e-mail send to the lot contact person. This can be turned on or off)

    >> Automatically e-mail winning bidders (who choose option)
    (This allows you to have the system automatically e-mail the winning bidders that have chosen to be e-mailed upon a winning bid.) (This feature can be turned on or off)

    >> Automatically e-mail losing bidders (who choose option)
    (This allows you to have the system automatically e-mail the losing bidders that have chosen to be e-mailed upon a losing bid.) (This feature can be turned on or off).

    SENDING E-MAILS: CF SCHEDULER VS EMAZE AUCTION SCRIPT

    In this section you can choose to use the Cold Fusion scheduler to send the automated e-mail in the auction system. If you choose this option, the Cold Fusion scheduler must be registered in the CF Administrator.

    EXTENDING CLOSING TIME IF NOT BIDS ARE RECEIVED FOR A LOT

    This section allows you to determine if you as the Admin want to extend lot closing times if no bids have been made in the lot by default.

    >> E-Mail Lot Contact Person
    This means that the system will e-mail the lot contact person telling them that at closing time there have been no bids. At this time, if so permitted, the lot contact person may go into their lot and extend the closing time.

    >> Extend Lot Closing time [1] [hours(s)], [3] times.
    This option lets you set a default extension time for all lots. The first field lets you determine an integer. The next field lets you define the first integer that you entered in the first field. This integer will be either a function of hours or days as represented in the second field. The third field lets you determine how many times you would like to extend the lot closing time in this fashion for a lot that does not receive any bids. At the end of the specified time here, each lot will close by default if not bid on.

    RESERVE PRICE OPTIONS

    The following section is devoted to allow you, the Admin to create and manipulate different options regarding how the reserve price is handled on your auction website.

    >> When reserve is met, e-mail lot contact person.
    (this simply is an option to e-mail the lot contact person when the reserve price for that lot has been met by a bidder) (this feature can be turned on or off).

    >> When reserve is met, email losing bidders to notify them
    (this is simply an option to e-mail the losing bidders for a lot when the reserve price is met) (this feature can be turned on or off)

    >> When reserve price is not met:

    E-Mail lot contact person:
    (this feature can be turned on or off)

    Extend lot closing time:
    (this allows you the Admin to determine a default extension time on lots that do not meet their reserve price. The first field has to be an integer, the second field defines the integer found in the first field as either hours or days. The third field is also an integer and defines how many times you want the time extension to apply if the reserve price is not met. (this feature can be turned on or off)

    E-Mail current winning bidders when extending lot closing time.
    (this feature can be turned on or off)

    E-Mail current losing bidders when extending lot closing time
    (this feature can be turned on or off)

    >> When reserve price is not met and lot is closed:

    E-Mail to winning bidders:

    >> Do not E-Mail
    (choosing this, the system will not e-mail winning bidders when the lot closes and the reserve price is not met)

    >> Send reserve winner e-mail
    (choosing this, the system will e-mail the highest bidder of the lot)

    >> Send normal loser e-mail
    (choosing this will send a losing bid e-mail to all those that bid in the lot)

    E-Mail to losing bidders:

    >> Do not E-Mail
    (choosing this, the system will not e-mail losing bidders)

    >> Send reserve loser e-mail
    (choosing this, the system will mail out a standard reserve loser e-mail to the losing bidders)

    >> Send normal loser e-mail
    (choosing this, the system will mail out a standard loosing bidder e-mail)

    To accept the changes you have made on this page, make sure to click on the "Update Options" button.

    Bid Options
    Back To The Top
    This page will allow you, as the Admin to set and change some of the bidding options for your auction website. Below is a detailed explanation of each of the sections found on this page:

    STRICT BID INCREMENT:
    Enforce increment as mandatory multiple rather than a minimum (if turned on this feature enforces bid incriments to be a multiple of the last bid incriment rather than a minimum integer increment value) (this feature can be turned on or off)

    AUTO BID:

    >> DO NOT ALLOW AUTOBID ON ANY LOTS
    (if this option is chosen, it automatically disallows bidders from setting auto-bid in any lot on your auction website)

    >> OPTIONAL AUTOBID ON ALL LOTS - with checkbox
    (if this option is chosen, all lots will now have an optional autobid feature that bidders can select using a checkmark box)

    >> OPTIONAL AUTOBID ON ALL LOTS - without checkbox
    (if this option is chosen, all lots will now have an optional autobid feauture, but the feature can not be selected using a checkmark box)

    >> ALL BIDS AUTOMATICALLY USE AUTOBID
    (if this option is chosen, all lots and all bids automatically use the auto-bid feature)

    FULL QUANTITY:

    >> Bidder has option of accepting full quantity only on multiple quantity lots. (this feature can be turned on or off)

    BID VERIFICATION:

    >> Do not verify bids before they are submitted
    (choosing this option will have the system perform no verification with the bidder to ensure it is the correct bid before submitting it to the system.)

    >> All bids must be verified before they are submitted
    (choosing this option will have the system perform a verification with the bidder to ensure it is the correct bid before submitting it to the system.)

    >> Bidder has option of verifying bid before it is submitted
    (choosing this option will allow the bidder to decide whether they want to verify their bid before submitting it to the system.)

    BID RESULTS:

    >> Display bid results after bid is proccessed
    (choosing this option, the system will show the bidder a seperate page with his or her bid results)

    >> Reload lot page after bid is processes
    (choosing this option, the system will keep the bidder on the same lot page and simply reload the page to reflect the results of his or her bid.)

    BID LOGIC:
    This is the way that winning bids are determined. You have three choices on how the bid logic will work.

    >> Maximum Bid, Bid Quantity, Bid Date/Time
    >> Maximum Bid, Bid Date/Time, Bid Quantity
    >> Maximum Bid, Bid Date/Time
